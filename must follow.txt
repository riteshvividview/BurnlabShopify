ARCHITECTURE RULES:

1. Section Scoping — MANDATORY

Assign a unique ID at the top:

liquid  {%- assign section_id = 'burnlab-[section-slug]-' | append: section.id -%}

ALL CSS must be scoped to #{{ section_id }} — no global selectors that could leak into other sections.
Begin every <style> block with a universal reset scoped to this section:

css  #{{ section_id }} *, #{{ section_id }} *::before, #{{ section_id }} *::after {
    box-sizing: border-box; margin: 0; padding: 0;
  }

Declare section-level CSS variables on #{{ section_id }} { } so schema color/size settings feed directly into CSS via Liquid output.

2. Liquid Variable Assignment

Assign ALL schema settings to Liquid variables at the top inside {%- liquid -%} tags before the <style> block.
Use | default: fallbacks on every variable so the section renders correctly even with no settings saved yet.

3. z-index Layering
Always stack layers in this order (low to high):

Background color / media
Glow blobs (z-index: 1)
Grain overlay (z-index: 2)
Decorative bg text (z-index: 2)
Content (z-index: 10)
Floating badges / decorative elements (z-index: 15)

4. overflow: hidden

The section root must always have overflow: hidden and position: relative; isolation: isolate;
Any absolutely-positioned decorative element (glow, blob, bg text) must NOT escape the section boundary.

On mobile, all CSS Grid layouts must collapse to grid-template-columns: 1fr.

ANIMATION RULES:

Think creatively for each section type. The animation style must match the section's purpose and feel unique, not generic. Do NOT use the same animation pattern for every section.
Examples of what already exists (do NOT repeat these):

Hero: dual-row scroll parallax background text sliding in opposite directions + mouse-move parallax on glows
Collection tabs: scroll reveal + tab ripple on click + 3D flip / slide-up panel transitions

For this section, invent an animation that fits its purpose. Think like a creative director:

What kind of content is in this section? (stats, features, testimonials, process steps, etc.)
What motion makes the content feel alive without distracting from it?
Could numbers count up? Could lines draw in? Could cards cascade? Could images breathe?
Use scroll-triggered IntersectionObserver for reveal animations — never auto-play animations that fire before the user sees the section.

Animation implementation rules:

ALWAYS use will-change: transform on animated elements.
ALWAYS respect prefers-reduced-motion:

css  @media (prefers-reduced-motion: reduce) {
    /* Disable all transitions, animations, parallax */
  }

Animate with transform and opacity only — never animate width, height, top, left, margin (causes layout thrash).
Never auto-play a looping animation that could distract on mobile.
Stagger reveal animations using the .bl-stagger class or transition-delay increments of 0.08s per child.

IntersectionObserver pattern (use this exact pattern):
javascriptconst observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('is-visible');
      // Unobserve after reveal for performance:
      observer.unobserve(entry.target);
    }
  });
}, { threshold: 0.15 });
document.querySelectorAll('#{{ section_id }} .bl-reveal, #{{ section_id }} .bl-reveal--left, #{{ section_id }} .bl-reveal--right, #{{ section_id }} .bl-reveal--scale').forEach(el => observer.observe(el));
